## C++简介
历史
    1979年，本贾尼-斯特劳斯特卢普 在C语言的基础上设计了一门新的编程语言，最初称为 C With Classes.
    1984年，正式命名为 c plus plus.
    C++兼容C语言的，现在一般称C和C++为一个系列的语言，用 c/c++ 进行描述。
     
领域
    C++主要应用于 系统软件、嵌入式、图形界面应用、后端(服务端)、游戏开发、科学计算等
    
标准
    1998年，发布了第一个世界标准：C++98/03
    2011年，发布了第二个世界标准：C++11
    再后来，每隔三年，发布一个新的标准：C++14、C++17、C++20、C++23、C++26
    如果在代码中使用了C++11及以上标准中的语言特性，在(交叉)编译时，可能需要加上编译选项 -std=c++11
    
环境
    在Ubuntu中，使用g++, 一般需要自行安装：
        sudo apt install g++

    在ARM-LINUX中，使用arm-linux-g++, 需要自己配置，方式与arm-linux-gcc类似。
        6818：5.4/5.5
        3568: 7.5

    在windows中，使用微软的IDE: visual studio 2026




## 第一个C++程序
```cpp
#include <iostream>

int main()
{
    std::cout << "hello cpp world" << std::endl;
    return 0;
}
```
说明：
1、标准C++库中的头文件没有扩展名称
    如果是使用标准C或操作系统的头文件，使用方式与C相同。
    如果是自已编写的头文件，还是要写扩展名，可以是 *.h , 也可以是 *.hpp
    iostream 是用于输入输出相关功能的头文件。

2、std 是C++标准库中一个作用域的名字，用于存放C++标准库中的各种名字，这种用来存放名字的作用域，称为 namespace(命名空间/名字空间)

3、:: 称为作用域运算符，用于访问指定作用域中的某个名字
    
4、标准IO流对象
    cin     标准输入流  stdin
    cout    标准输出流  stdout
    cerr    标准错误流  stderr
    
5、输入输出运算符
    <<  输出运算符  结合cout或cerr使用
    >>  输入运算符  结合cin使用

6、endl 指 end line, 相当于 '\n'


## 命名空间 namespace 
namespace 是一种作用域，是对全局作用域的切分，用于存放各种名字，如变量名、数组名、函数名、类型名、命名空间名称等
namespace 适用于大型软件的开发，用于解决软件开发过程中存在的【名字冲突】的问题。
同时也表示一个逻辑单元，使得在用户看来，它们是一个整体。


定义：
```
namespace 命名空间名称
{
    定义变量
    定义数组
    定义函数
    定义类型
    定义命名空间
    ...
}
```
使用方式：
1、显示限定
    命名空间名称::成员名称
    例：
    std::cout
    std::cin
    std::endl
    china::print

2、using 声明
    using 命名空间名称::成员名称;
    例：
    using std::cout;
    using std::endl;
    cout << "hello";
    cout << "cpp";
    cout << "world";
    cout << endl;

3、using 指示
    using namespace 命名空间名称;
    例：
    using namespace std;
    using namespace china;
    cout << a << endl;


说明：
1、全局作用域也可以理解为一个命名空间，但全局作用域没有名字
    如果需要显示的使用 全局作用域中的成员，方式如下：
    ::全局的名字
    例：
    int fd = ::open("1.cpp", O_RDONLY);
    ::close(fd);

2、using除了可以用来说明命名空间，还可以用来 定义 类型别名(c++11)。
一般格式：
    using 类型别名 = 类型名;
    例：
    using size_t = unsigned long;
    using uchar = unsigned char;
    using u32 = unsigned int;
    using PF = void(*)();
    
    void print();
    PF pf1 = print;


## 布尔类型 bool
bool类型是C++的基础(内置)类型之一，用于表示逻辑概念：
    零为假，非零即为真。

在C++中，用true表示真，用false表示假。
bool类型只有两种值：true和false
但bool类型变量可接收任意类型的值，例：
```cpp
bool ok;
ok = true;
ok = false;
ok = 100;           // true
ok = 1-2;           // true
ok = 3.14;          // true
ok = "hello";       // true
ok = NULL;          // false
```

bool类型主要用于以下场合：
1、条件判断
2、函数返回

例：给定一个字符串，输出其中的数字字符
    "hello cs2607F" => "2607"
    把判断某个字符是否为数字的功能 包装成函数：
    bool isdigit(char c);
    假设函数定义完成，可以调用
    bool ok = isdigit('a');
    if (ok)  // ok == true
    {

    }
    if (!ok) // ok == false
    {

    }

练习：
    给定一个字符串，输出其中的数字字符




## 引用 
引用是一个已存在对象的别名。
一般格式：
```
类型名 & 引用名 = 对象名;
```
例：
```cpp
int a = 100;
int & r = a; // r是a的别名，r就是a, a就是r
```

基本特点：
1、定义引用时，必须初始化
2、引用一旦定义完成，就不能再成为其他对象的别名
3、对引用进行的任何操作，都是在操作与之绑定的那个对象
4、引用是别名，不是对象(对象需要占用存储空间，引用不需要)

引用最主要用的用途，是用于函数传参
C++中，函数传参的方式：
    传值
    传地址
    传引用

引用分类：
1、常规引用(左值引用)
    只能绑定到左值的引用
    例：
    int a = 100;
    int & r1 = a;
    int & r2 = 100; // error

2、const引用
    const引用既能绑定到左值，也能绑定到右值，不能通过const引用修改它所绑定对象的值
    例：
    int a = 100;
    const int& r1 = a;      // 绑定到左值
    cosnt int& r2 = 100;    // 绑定到右值


练习：
    编写算法swap，实现交换两个整型变量的值
    要求用指针和引用 分别实现。
    int a = 1;
    int b = 2;
    swap(&a, &b);
    swap(a, b);



## 函数重载
所谓函数重载，指在相同的作用域中，定义相同名字的函数，但这些函数的参数列表不相同。
例：
```cpp
void swap(int* p1, int* p2); // 1
void swap(int& r1, int& r2); // 2
```
在调用这些重载函数时，编译器会根据实参，自动匹配对应的函数。
如果实参的类型与形参 不完全匹配，则编译器会自动尝试类型转换(不保证成功).
例：
```cpp
int a = 1;
int b = 2;
swap(a, b);     // 调用swap2
swap(&a, &b);   // 调用swap1

double d = 3.1;
swap(a, d);

```
说明：
1、函数重载要求参数列表不相同，具体如下：
    要么 个数不同
        int sum(int a, int b);
        int sum(int a, int b, int c);
    要么 类型不同
        int sum(int a, int b);
        double sum(double a, double b);
    要么 顺序不同
        int sum(int a, double b);
        int sum(double a, int b);

2、函数重载与返回类型无关



练习：
    编写算法compare, 实现整数，小数、字符串的比较
    如果第一个参数小于第二个参数，则返回-1
    如果第一个参数大于第二个参数，则返回1
    否则 ，返回0 

    编写算法find, 实现对整数数组、浮点数数组、字符串数组的查找
        查找特定的值，找到则返回其下标，否则返回-1

## 默认参数
在C++中，声明函数时，如果函数有形参，则允许给形参设置一个默认值。
例：
```cpp
void foo(int x=0);
```
当调用这种函数时，如果不传递实参，则参数自动使用声明时设置的默认值。
```cpp
foo();  // x == 0
foo(1); // x == 1
```
一般格式如下：
```
返回类型 函数名(类型 形参名=默认值, ...);
```
说明：
1、如果函数的声明与定义分开，则默认值只需要写在声明处，定义时不用再写。
2、如果函数有多个形参，给其中某个形参设置了默认值，则它右边所有的形参都必须设置默认值。


练习：
    pow(3) => `3*3`
    pow(3,3) => `3*3*3`



## 内联函数 inline
用关键字 inline 修饰说明的 函数，称为内联函数。
一般格式：
```
inline 返回类型 函数名称(参数列表)
{
    函数体
}
```
在C++中，inline 是一种优化手段，用于减少函数调用的开销，以提高程序的执行效率。
当函数被说明为内联函数时，编译器在编译阶段 可能 会把函数体 直接复制到函数调用点，而不是进行常规的函数调用。
这样可以消除函数调用的开销，特别是那些函数体简短且需要频繁调用的函数。
内联函数可能会增加最后生成的可执行文件的大小。

说明：
1、函数即使说明为内联函数，但能否内联成功，取决于编译器
2、在struct/class中，定义的函数，默认就是内联函数。



## 关键字 extern "C"
extern "C"称为 链接指示，用于指示C++编译器 按照C语言的规则 去编译和链接指定的函数。

C++编译器在编译函数时，会修改函数名称(主函数例外)
    给函数名添加前缀和后缀，前缀一般表示作用域、函数名长度等，后缀一般表示参数类型
    例：
    int sum(int a, int b);           
    double sum(double a, double b);  
    以上两个函数被C++编译器编译后，函数名被修改为：_Z3sumii 和 _Z3sumdd

但C语言编译器从不修改函数名。
所以，当在C++中，调用C库中的函数时，就可能会出现 由于函数名不匹配 而造成的 链接错误。

解决方案：使用关键字 extern "C" 对C库中的函数进行说明。

典型示例：
```cpp
#ifdef __cplusplus
extern "C" 
{
#endif

    int sum(int a, int b);
    
#ifdef __cplusplus    
}
#endif
```
说明：
1、__cplusplus是C++编译器内置的一个宏，用于C和C++编译器
2、extern "C"说明的函数，不能重载。
