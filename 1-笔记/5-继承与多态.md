## 继承
所谓继承，指的是在已有类型的基础上，创建新的类型，新的类型会继承原有类所有的属性和行为(扩展已有类型)。
继承能实现 代码复用、减少代码冗余、提高开发效率等。
继承主要用于 描述那些非常相似但有细微差异的类型之间的关系。
例：
    计算机
        台式机、笔记本、平板、手机

    学生
        大学生、中学生、小学生

    车
        小汽车、公交车、洒水车

用于描述共性的类型，称为基础类型，简称为基类(Base Class), 也叫父类，如上述示例中的 计算机、学生、车
在基类的基础上创建出来的新类型，称为派生类(Derived Class), 也叫子类，如上述示例中的 笔记本、大学生、小汽车

在C++中继承的语法如下：
```
class/struct 派生类名称: 继承方式 基类名1, ..., 继承方式 基类名n
{
    派生类新增加的属性和行为
};
```
根据基类的数量，继承可分为：
单(一)继承：只有一个基类
多(重)继承：有两个或两个以上的基类

继承方式，决定了基类中的成员，在派生类中的可见性(访问权限)，分三种：
public      公有继承
private     私有继承
protected   保护继承

说明：继承方式可以省略不写(但不建议)，如果省略不写，则
    class 默认为 私有继承， struct 默认为 公有继承。


继承方式具体规则：
public公有继承
    基类的公有成员，通过公有继承，成为派生类自己的公有成员。
    基类的私有成员，通过公有继承，成为派生类不可直接访问的成员(派生类内部和外部都不能直接访问)。
    基类的保护成员，通过公有继承，成为派生类自己的保护成员(派生类内部可直接访问、外界不能直接访问)。


private私有继承
    基类的公有成员，通过私有继承，成为派生类自己的私有成员。
    基类的私有成员，通过私有继承，成为派生类不可直接访问的成员(派生类内部和外部都不能直接访问)。
    基类的保护成员，通过私有继承，成为派生类自己的私有成员。    

protected保护继承
    基类的公有成员，通过保护继承，成为派生类自己的保护成员。
    基类的私有成员，通过保护继承，成为派生类不可直接访问的成员(派生类内部和外部都不能直接访问)。
    基类的保护成员，通过保护继承，成为派生类自己的保护成员。    



## 继承中的构造与析构
构造函数与析构函数 不能继承。
基类成员的初始化 由基类的构造函数负责，基类资源的释放由基类的析构函数负责。

如果基类有默认构造函数，当实例化派生类对象时，会自动调用基类的默认构造函数。
如果基类没有默认构造函数，或需要调用其他带参数的构造函数，
    则必须在派生类的构造函数初始化列表中 显示 调用基类的构造函数。

如果派生类有成员对象，则派生类对象创建时，构造函数的调用顺序如下：
    基类的构造函数
    成员对象的构造函数
    派生类的构造函数
当派生类对象销毁时，析构函数的调用顺序如下：
    派生类的析构函数
    成员对象的析构函数
    基类的析构函数

例：
```cpp
class Base 
{
public:
    Base()
    {
        x = 0;
        y = 0;
        z = 0;
        cout << "Base()" << this << endl;
    }
    Base(int x, int y, int z)
    {
        this->x = x;
        this->y = y;
        this->z = z;
        cout << "Base(int,int,int)" << endl;
    }
    ~Base()
    {
        cout << "~Base() " << this << endl;
    }
    int x;
private:
    int y;
protected:
    int z;
};

class Derived: public Base 
{
public:
    Derived()
    {
        cout << "Derived() " << endl;
    }
    Derived(int x, int y, int z): Base(x,y,z), a(0)
    {
        cout << "Dervied(int,int,int)" << endl;
    }
     ~Derived()
    {
        cout << "~Derived() " << this << endl;
    }

    void setValue()
    {
        x = 1;
        // y = 2; // 在类的内部无法直接访问基类的私有成员
        z = 3;
    }

private:
    int a;
    Base b; // 成员对象
};


int main(int argc, char const *argv[])
{
    Derived d1; // Base() Base() Derived()
    
    Derived d2(1,2,3); // Base(int,int,int) Base() Derived(int,int,int)
    return 0;
}
```

## 继承中的同名成员
派生类可以在基类的基础上 添加新的成员，这些新的成员【可以】与基类的成员同名(因为基类与派生类的作用域不同)。
如果派生类的成员与基类的成员 同名，则会出现 成员隐藏的现象(基类的同名成员会被隐藏)。

函数隐藏：
    在派生类中定义与基类同名的成员函数，通过派生类对象调用这种同名成员函数时， 默认调用的是派生类新增加的同名成员函数。
    基类的同名成员函数 被隐藏。

说明：
    如果需要调用那些被隐藏的成员，只需要指定作用域即可。
    函数隐藏出现在继承的层次结构中。
    函数隐藏只与函数名相关，和函数的参数列表、返回类型 无关。

例：
```cpp
class Base 
{
public:
    void setX(int x)
    {
        this->x = x;
        cout << "Base::setX()" << endl;
    }
    int getX() const 
    {
        return x;
    }
protected:
    int x;
};


class Derived: public Base 
{
public:
    void setX(int x)    // 同名函数
    {
        this->x = x;
        cout << "Derived::setX()" << endl;
    }
    int getX() const    // 同名函数
    {
        return x;
    }
protected:
    int x;              // 同名变量
};

int main(int argc, char const *argv[])
{
    Derived d1;
    d1.setX(100);                   // 调用的是派生类新增加的同名成员
    cout << d1.getX() << endl;      // 调用的是派生类新增加的同名成员
    cout << d1.Base::getX() << endl;// 调用基类的同名成员
    return 0;
}
```


## 公有派生类与基类的关系
类与类之间的关系，一般讨论两种：
组合，也叫做`has-a`("有一个")，如 计算机 有CPU、内存、主板、硬盘等
继承，也叫做`is-a`("是一个")，如 笔记本 是 计算机，手机 是 计算机等

由于公有派生类与基类存在这种is-a的关系，所在，在程序中需要基类的时候，都可以用它的任意公有派生类来代替。
具体如下：
1、用派生类对象 初始化 基类对象
```cpp
    Derived d1;
    Base b1 = d1;
```

2、用派生类对象给基类对象赋值
```cpp
    b1 = d1;
```

3、基类指针 可以指向它的任意公有派生类对象
```cpp
    Base* p1 = new Derived;
```

4、基类引用 可以绑定到它的任意公有派生类对象
```cpp
    Base& r = d1;
```

注：
    基类指针可以指向派生类对象，但不能通过基类指针直接访问派生类新增加的成员。
    引用同样如此。



## 多重继承
所谓多重继承，指一个派生类有两个或两个以上基类的情况，派生类会继承它所有基类的属性和行为。
例：
```cpp
class Base1 {};
class Base2 {};
class Derived: public Base1, public Base2 {};
```
多重继承中的二义性问题：
    派生类继承的多个基类中有同名成员，在派生类中访问这些同名成员时，会出现模糊不清的情况。
解决方案：  
    显示的指定作用域即可。
例：
```cpp
class Base1
{
public:
    void print() const
    {
        cout << "Base1.x = " << x << endl;
    }

private:
    int x;
};

class Base2
{
public:
    void print() const
    {
        cout << "Base2.x = " << x << endl;
    }

private:
    int x;
};

class Derived : public Base1, public Base2
{
};

int main(int argc, char const *argv[])
{
    Derived d1;
    cout << sizeof(d1) << endl;
    // d1.print(); 编译出错
    d1.Base1::print();
    d1.Base2::print();
    return 0;
}
```
多重继承中的"菱形继承(钻石继承)"：
例：
```cpp
class A {};
class B: public A {};
class C: public A {};
class D: public B, public C {};
```
菱形继承存在的问题：
    最顶层的A类中的成员，通过多条继承路线，到达最底层的D类时，会出现多份拷贝，且被初始化多次。

解决方案：虚继承
    在菱形继承中，B类和C类在继承A类时，在它们的继承方式左边(或右边)，加一个关键字`virtual`,这种继承称为 虚继承。
    虚继承能保证A中的成员，在派生类D中，只保留一份，且只初始化一次(由D类调用A的构造函数)。

```cpp
class A {};                     // 虚基类
class B: virtual public A {};   // 虚继承
class C: virtual public A {};   // 虚继承
class D: public B, public C {};
```

说明：
    虚继承 虽然解决了虚基类成员被复制多次的问题，但需要付出一定的代价：用一次virtual，就会多出一个虚指针，虚继承产生的虚指针，会被派生类继承。
    当计算派生类对象大小时，需要考虑这些隐藏的虚指针 及 字节对齐。
