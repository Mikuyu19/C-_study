## 面向过程和面向对象
C++是一门面向对象的编程语言
面向对象和面向过程 是解决问题的两种方法(思想)


面向过程
面向过程主要关注的是解决问题的步骤(过程),以过程为核心，强调事件的流程与顺序。
面向过程将复杂的问题分解为一系列的函数调用，这些函数按照一定的顺序执行，共同完成任务。
面向过程的程序通常将数据与相关的操作分离，函数通常操作外界传入的数据。
例：在开发板的显示屏幕上绘制一个矩形
1、打开屏幕文件     open
2、内存映射         mmap
3、绘制像素点       draw_point
4、解除映射         munmap
5、关闭文件         close

面向对象
面向对象主要关注的是解决问题所需要的对象，它将问题分解为一系列的对象。
面向对象中 数据与相关的操作 封装在一起，形成对象，对象内部的方法操作自己的数据。
每个对象都有自己的属性(数据)和行为(函数)。
例：在开发板的显示屏幕上绘制一个矩形
1、定义屏幕类型，描述屏幕的属性和行为
2、实例化屏幕对象
3、定义矩形类型，描述矩形的属性和行为
4、实例化矩形对象
5、给矩形发消息，绘制到指定的屏幕上


面向对象的三大核心特征：
封装 Encapsulation
继承 Inherit
多态 Polymorphism


## 面向对象编程(OOP Object-Oriented Programming)
封装 是指将对象的属性和行为包装在一起，对外隐藏对象的内部实现，只公开有限的接口供外界使用。
封装作用：
    数据保护：防止外界直接访问内部的数据，从而维护数据的完整与正确。
    模块化：通过封装，类的内部实现可以和接口分离，使得类的设计更加简单。
    灵活性：内部实现可以变化，而不影响外界的使用。


类(class)
    类就是类型，是自定义的数据类型。
    用于描述实现世界中那些复杂的概念，描述同一类事物的属性和行为。
    例：
    矩形
        属性：宽、高、位置、颜色等
        行为：绘制、计算面积、计算周长等

    学生
        属性：学号、姓名、年龄、性别、学校、专业等
        行为：学习、说话、睡觉、运行等
    
    封装通过访问权限控制来实现对象数据的隐藏和保护
    访问权限分三种：
    public      公有    表示成员可以被类的外部直接访问
    private     私有    表示成员只能在类的内部访问，外界不能直接访问
    protected   保护    表示成员只能在类的内部和派生类访问，外界不能直接访问

    类的一般形式：
    class/struct 类名
    {
    访问权限符:
        属性
        行为
    };
    说明：
    对于class来说，如果成员没有明确的标明访问权限，则默认为 private
    对于struct来说，如果成员没有明确的标明访问权限，则默认为 public


对象(object)
    从面向对象的角度，一切皆对象。
    从编程的角度来说，类 类型的变量，就是对象。
    用类类型定义变量，称为实例化对象。
    一般形式：
    类名 对象名;



## 类的组织形式
标准C++中的类，一般由两部分组成
类的声明：一般写在头文件中
类的定义：一般写在源文件中
例：
```cpp
// 类的声明
class Point
{
public:
    void setX(int x);
    void setY(int y);
    void move(int x, int y);

    int getX() const;
    int getY() const;

private:
    int _x;
    int _y;
};
```

```cpp
// 类的定义
#include "point.hpp"

void Point::setX(int x)
{
    _x = x;
}

void Point::setY(int y)
{
    _y = y;
}

void Point::move(int x, int y)
{
    _x = x;
    _y = y;
}

int Point::getX() const
{
    return _x;
}

int Point::getY() const
{
    return _y;
}
```
说明：在类外定义成员函数时，必须指明作用域

```cpp
// 类的使用
#include "point.hpp"
#include <iostream>
using namespace std;

void print_point(const Point& pos)
{
    cout << pos.getX() << endl;
    cout << pos.getY() << endl;
    // pos.setX(100); // error！ pos是const引用，只能调用只读成员函数
}

int main(int argc, char const *argv[])
{
    Point pos;
    pos.move(100, 200);
    print_point(pos);
    return 0;
}
```



## 只读成员函数
对于对象的属性，一般会提供一组函数进行访问(access function)
一般用get函数来读取成员的值，用set函数来修改成员的值
对于那些确实只读取成员值的函数，可以在函数的参数列表后面加一个关键字 const, 称为只读成员函数
一般如下：
```
class 类名
{
public:
    返回类型 函数名(参数列表) const ; 
};

返回类型 类名::函数名(参数列表) const 
{
    函数体
}
```
说明：
在成员函数声明时添加了const, 定义时也必须添加
如果在这种只读成员函数内部修改了成员的值，则编译出错。
通过const引用或指向const的指针，只能调用这种只读成员函数。
例：
```cpp
void print_point(const Point& pos)
{
    cout << pos.getX() << endl;
    cout << pos.getY() << endl;
    // pos.setX(100); // error！ pos是const引用，只能调用只读成员函数
}

int main(int argc, char const *argv[])
{
    Point pos;
    pos.move(100, 200);
    print_point(pos);
    return 0;
}
```

## 构造函数
构造函数是类中的特殊成员函数
特点：
    构造函数用于对象成员的初始化
    构造函数的名字与类名相同
    构造函数没有返回类型，只有函数名、参数列表及函数体
    构造函数在实例化对象时，自动调用。
    如果类中没有显示的定义构造函数，编译器会自动生成一个默认构造函数。
例：
```cpp
class Point
{
public:
    Point();                // 构造函数1
    Point(int x, int y);    // 构造函数2，重载
};

Point p1;           // 自动调用构造函数1
Point p2(100,200);  // 自动调用构造函数2
Point p3{1,2};
```
说明：
1、默认构造函数，指创建对象时不提供实参，就能调用的构造函数，有两种情况：
    没有参数，如 Point()
    有参数，但全部有默认值，如 Point(int x=0, int y=0);

2、如果在实例化对象时，需要传递参数进行初始化，常见形式：
    类名 对象名(实参, ...);
    类名 对象名{实参, ...}; // C++11
    注：如果实例化对象时，不传递参数，要么不写圆括号，要么用大括号
    例：
    Point p4();     // 这是一个函数声明，p4是函数名
    Point p5{};     // 这是对象实例化，p5是Point类型的对象
    
3、匿名对象(临时对象)
实例化对象时，一定会调用构造函数
直接调用构造函数，一定会实例化一个对象，这种对象没有名字，生命周期短暂，一般称为临时对象(匿名对象)
匿名对象最常用于函数传参。
例：
    foo(Demo(300)); // 使用临时对象做为参数



## 构造函数的成员初始化列表
所谓成员初始化列表，指在定义构造函数时，在参数列表的圆括号后面、在函数体的大括号前面，用一个冒号引导，进行成员的初始化，一般如下：
```
构造函数名(参数列表): 成员变量1(初值), ..., 成员变量n(初值)
{
    函数体
}
```
以下三种情况，必须使用成员初始化列表：
1、成员是常量
2、成员是引用
3、成员对象没有默认构造函数
例：
```cpp
class Demo 
{
public:
    Demo(int x, int y): _x(x), _y(y), _r(_x), _oth(100) {}
private:
    int _x;
    const int _y; // 成员是常量
    int& _r;      // 成员是引用
    Other _oth;   // 成员对象没有默认构造函数
};
```
建议使用成员初始化列表，效率更高，代码更简洁。



练习：
    在开发板显示屏幕上绘制一个圆形
    Screen s("/dev/fb0");
    Circle c(20);
    c.draw(s);

## 关键字 explicit
explicit多用于修饰说明构造函数，用于阻止编译器 使用构造函数 进行隐式类型转换。
例：
```cpp
class Demo 
{
public:
    Demo(int x):_x(x) {}    // 1
    Demo(const char* s) {}  // 2

    void print() const 
    {
        cout << "x = " << _x << endl;
    }
private:
    int _x;
};


int main(int argc, char const *argv[])
{
    Demo d1(100);  // 显示调用构造函数1
    d1.print();

    Demo d2{200};  // 显示调用构造函数1
    d2.print();

    Demo d3 = 300; // 隐式转换： Demo d3 = Demo(300)
    d3.print();
    
    Demo d4 = "hello"; // 隐式转换： Demo d4 = Demo("hello")
    return 0;
}
```
如果在构造函数前面加上关键字 `explicit`, 就能阻止编译器的隐式转换。
例：
```cpp
class Demo 
{
public:
    explicit Demo(int x):_x(x) {}    // 1
    explicit Demo(const char* s) {}  // 2

    void print() const 
    {
        cout << "x = " << _x << endl;
    }
private:
    int _x;
};


int main(int argc, char const *argv[])
{
    Demo d1(100);  // 显示调用构造函数1
    d1.print();

    Demo d2{200};  // 显示调用构造函数1
    d2.print();

    Demo d3 = 300; // 编译出错，不允许隐式转换
    d3.print();
    
    Demo d4 = "hello"; // 编译出错，不允许隐式转换
    return 0;
}
```


## 析构函数
析构函数也是类中的特殊成员函数
特点：
    析构函数用于释放对象所占用的系统资源。
    析构函数的名称 由 波浪线加类名组成： ~类名
    析构函数没有返回类型，只有函数名、参数列表及函数体，且参数列表必须为空。
    对象销毁时，自动调用。
    如果类中没有显示的定义析构函数，编译器自动生成一个析构函数。
例：
```cpp
class Demo 
{
public:
    Demo() // 默认构造函数
    {
        cout << "Demo()" << endl;
    };

    ~Demo() // 析构函数
    {
        cout << "~Demo()" << endl;
    }
};

int main(int argc, char const *argv[])
{
    Demo d1; // 创建对象，自动调用构造函数，对象销毁时，自动调用析构函数
    return 0;
}
```

练习：
有如下代码：
```cpp
int main()
{
    cout << "cpp" << endl;
    return 0;
}
```
在不修改主函数的情况下，使得以上代码输出如下：

cpp
world


## 动态内存
C++中，对象所占用的内存，分三种：
局部对象            栈区(.stack)
全局对象/静态对象    静态存储区(.data .bss)
动态对象            堆区(.heap)

C语言中，使用标准库函数 malloc/free 系列函数 来管理动态内存。
C++中，如果处理的是基础类型的数据，可以继续使用 malloc/free
但如果处理的是类类型的对象，则不再建议使用 malloc/free

在C++中，使用new/delete运算符(关键字) 来管理动态内存。
new用来分配动态内存，delete用来释放动态内存。

语法格式：
    类型名* 指针名 = new 类型名;
    类型名* 指针名 = new 类型名(初值);
    类型名* 指针名 = new 类型名{初值}; // C++11
    类型名* 指针名 = new 类型名[元素个数];
    类型名* 指针名 = new 类型名[元素个数]{初值列表};

    delete 指针名;      // 释放单个对象
    delete []指针名;    // 释放对象数组

例：
```cpp
int* p1 = new int;
Demo * p2 = new Demo;

int* p3 = new int(100);
Demo *p4 = new Demo(200);

int* p5 = new int[5]{1,2,3,4,5};
Demo* p6 = new Demo[5];
Demo* p7 = new Demo[5]{Demo(1), Demo(2), Demo(3)};

delete p1;
delete p2;
delete p3;
delete p4;
delete []p5;
delete []p6;
delete []p7;
```

空指针说明：
传统的宏 NULL， 在C++中可以继续使用
但是NULL在某些特定的情况下，可能会出现语法模糊的现象：
void foo(int x);    // 1
void foo(int* p);   // 2

foo(NULL); // 编译出错，因为以上两个函数都可以，编译器无法选择

从C++11起，引入了一个字面值常量 nullptr 专门用于表示空指针。
nullptr不是宏，它有自己专门的类型 nullptr_t

foo(nullptr); // 精确匹配第2个foo函数。




## 字符串类型 string
C++标准库中提供了一个专门的类类型 string, 用于表示字符串这个概念。
使用方式：
1、包含头文件
`#include <string>`

2、实例化对象

c_str() 把string转换成const char*
find()  从字符串中查找指定的子串，找到则返回其下标，否则返回 std::string::npos

## 异常
异常是C++中的一种错误处理机制。
在编程过程中，可能会碰到两种错误：
1、编译错误(语法错误)
2、运行时错误(异常)


C++中的异常由两部分组成：
1、抛出异常
使用关键字throw 来抛出异常，表示程序出现了运行时错误。
语法格式：
    throw 表达式;
例：
```cpp
void foo(int x)
{
    if (x < 0)
        throw 1; // 抛出整型异常
    cout << "x = " << x << endl;
}
```
如果程序执行了throw语句，程序流就不再往下执行，而是从当前函数开始查找异常处理语句
如果异常没有被处理，则往上抛出，抛出给当前函数的调用者，直到被处理为止
如果异常被系统捕获，则终止当前进程。


2、捕获处理异常
使用try/catch语句 来捕获处理异常。
    try
    {
        可能抛出异常的代码块
    }
    catch(异常类型1 形参)
    {
        处理异常类型1
    }
    catch(异常类型n 形参)
    {
        处理异常类型n
    }
    catch(...)
    {
        处理任意类型的异常
    }

如果try中的语句抛出了异常，则程序流从try中抛出异常的位置 跳转到catch语句，进行异常类型匹配。
如果匹配成功，则进入catch中处理异常。
如果匹配失败，则按顺序往下匹配。
如果所有的catch都匹配失败，则继续往上抛出，直到被处理为止。
