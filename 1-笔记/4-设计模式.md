## 设计模式
所谓设计模式，是面向对象程序设计中最有价值的经验总结，并且用简洁可复用的形式表达出来。
每个模式描述了一个在我们周围不断重复发生的问题，以及该问题的核心解决方案。这样就能一次又一次的使用该方案而不必做重复劳动。

Gof设计模式收录了23种设计模式，根据功能，分为三类：
创建型模式
    抽象工厂模式、生成器模式、工厂方法模式、原型模式、单件(单例)模式。

结构型模式
    适配器模式、桥接模式、组成模式、装饰器模式、外观模式、享元模式、代理模式。

行为型模式
    职责链模式、命令模式、解释器模式、迭代器模式、中介者模式、备忘录模式、观察者模式、状态模式、策略模式、模板方法、访问者模式。



## 单件(单例)模式
保证一个类仅有一个实例，并提供一个访问该实例的全局访问点。
解决方案：
1、让类自身负责保存它的唯一实例。
2、这个类可以保证没有其他实例可以被创建(通过截取创建新对象的请求)。
3、并且它可以提供一个访问该实例的方法。

说明：
根据这个唯一实例的创建时机，可以把单例的实现分为两种：
1、惰性初始化(懒汉式)：直到第一次调用instance()函数时，才会创建这个唯一的实例。
    确保资源不会浪费，但第一次调用时效率略低。同时在多任务环境下可能不安全(C++11起，如果使用静态局部变量，则能确保线程安全)。

2、预先初始化(饿汉式)：程序运行时自动创建唯一实例。
    资源可能浪费，第一次获取唯一实例时效率较高。但是这个唯一实例的创建时机不确定，只能保证在主函数运行前被创建。

例：
```cpp
class Singleton
{
public:
    // 公有的静态成员函数
    static Singleton* instance()
    {
        // 静态局部对象,从C++11起，静态局部对象保证线程安全
        static Singleton _instance;

        return &_instance;
    }
    void setValue(int x)
    {
        this->x = x;
    }
    int getValue() const 
    {
        return x;
    }

private:
    // 隐藏构造函数
    Singleton() {}
    
    // 删除拷贝操作
    Singleton(const Singleton&) = delete;
    Singleton& operator=(const Singleton&) = delete;
    
    int x;
};
```

    把Screen类型，改写成单例类型，并测试使用。



## 迭代器模式    
提供一种方法顺序访问一个聚合对象中各个元素，而又不暴露该对象的内部表示。
核心思想：
    将聚合对象的访问与遍历 从聚合对象中 分离出来放入一个迭代器对象中。
    迭代器类定义了一组访问该列表元素的接口，迭代器对象负责跟踪当前的元素。

C++标准库的一般做法：
1、每个聚合对象都提供自己的迭代器(一般是嵌套定义)
2、迭代器提供一组简单但通用的接口，如 * ++  !=
3、每个聚合对象还需要提供两个专门的接口：
    begin() 用于获取聚合对象首元素的迭代器
    end()   用于获取聚合对象的 尾后迭代器(最后一个元素后面的那个位置)

例：
```cpp
#include <iostream>
#include <string.h>
using namespace std;

namespace china
{
    class string
    {
    public:
        string(const char *s = NULL)
        {
            if (s == NULL)
            {
                _data = new char[1];
                _data[0] = '\0';
            }
            else
            {
                _data = new char[strlen(s) + 1];
                strcpy(_data, s);
            }
        }
        // ....

        class iterator
        {
        public:
            iterator(char *p) : p(p) {}
            char &operator*()
            {
                return *p;
            }
            iterator& operator++()
            {
                ++p;
                return *this;
            }
            bool operator!=(const iterator& rhs) const 
            {
                return this->p != rhs.p;
            }

        private:
            char *p;
        };
        iterator begin()
        {
            return iterator(_data);
        }
        iterator end()
        {
            return iterator(_data + strlen(_data));
        }

    private:
        char *_data;
    };
}

int main(int argc, char const *argv[])
{
    china::string s1 = "hello world";

    for (china::string::iterator it = s1.begin(); it != s1.end(); ++it)
    {
        cout << *it << ' ';
    }
    cout << endl;
    return 0;
}
```


## 适配器模式 Adapter
将一个类的接口转换成客户希望的另外一个接口，使得原本由于接口不兼容而不能一起工作的类可以一起工作。
适配器也叫 包装器。

适配器根据实现方式可以分为：
对象适配器，用组合的方式实现，适配器内部包装一个对象，所有功能由这个内部的对象来完成。
类适配器，用继承的方式实现。

例：可以对链表类型进行包装，以适配出来栈和队列。
```cpp
class LinkStack
{
public:
    void push(std::string s)
    {
        list.push_back(s);
    }
    void pop()
    {
        list.pop_back();
    }
    std::string top() const 
    {
        return list.back();
    }
    bool empty() const
    {
        return list.empty();
    }

private:
    List list;
};

int main(int argc, char const *argv[])
{
    LinkStack s1;
    s1.push("hello");
    s1.push("cpp");
    s1.push("world");
    s1.push("!!!");

    while (!s1.empty())
    {
        cout << s1.top() << endl;
        s1.pop();
    }
    return 0;
}
```
